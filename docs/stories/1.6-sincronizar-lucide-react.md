# Story 1.6: Sincronizar Versões do lucide-react — Brownfield Addition

## User Story

Como desenvolvedor,
Eu quero versões consistentes do `lucide-react` em todos os pacotes,
Para evitar conflitos de dependência e comportamento inesperado.

## Story Context

**Existing System Integration:**

- **Integrates with:** `packages/ui/package.json` e `apps/web/package.json`
- **Technology:** npm/pnpm/yarn (Bun workspaces)
- **Follows pattern:** Catalog de versões no root `package.json`
- **Touch points:** `package.json` (root catalog), `packages/ui/package.json`, `apps/web/package.json`

## Acceptance Criteria

**Functional Requirements:**

1. `lucide-react` movido para o catalog no root `package.json` com versão única
2. `packages/ui` e `apps/web` referenciam a versão do catalog (`workspace:^` ou catalog)
3. Versão escolhida: `1.8.0` (mais recente, usada pelo web) ou `0.546.0` (usada pelo ui — verificar breaking changes)
4. Se breaking changes entre `0.546.0` e `1.8.0`, atualizar imports no `packages/ui`
5. `bun install` executa sem conflitos

**Integration Requirements:**

6. Todos os ícones usados continuam funcionando
7. Bundle size não aumenta significativamente

**Quality Requirements:**

8. Build passa sem erros (`bun run build`)
9. TypeScript check passa (`bun run check-types`)

## Technical Notes

- **Integration Approach:** Adicionar `lucide-react` ao `"catalog"` no root `package.json` com versão `1.8.0`, remover versões específicas de `packages/ui` e `apps/web`.
- **Existing Pattern Reference:** Root `package.json` já tem `"catalog"` com `react@^19.2.6`, `zod@^4.1.13`, etc.
- **Key Constraints:**
  - Verificar breaking changes entre `0.546.0` e `1.8.0` no changelog do lucide
  - `packages/ui` pode precisar de atualização de imports se houver mudanças de API
  - `bun install` com `linker = "isolated"` resolve versões do catalogo

## Definition of Done

- [x] `lucide-react` no catalog do root `package.json`
- [x] `packages/ui/package.json` e `apps/web/package.json` atualizados
- [x] `bun install` bem-sucedido
- [x] `bun run build` sem erros
- [x] `bun run check-types` sem erros

## Risk and Compatibility Check

- **Primary Risk:** Breaking changes entre `0.546.0` e `1.8.0` (major version jump)
- **Mitigation:** Verificar changelog do lucide-react entre as versões; preparar para atualizar imports se necessário
- **Rollback:** Reverter mudanças nos package.json e rodar `bun install`
