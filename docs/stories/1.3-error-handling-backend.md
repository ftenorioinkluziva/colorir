# Story 1.3: Error Handling Global no Backend — Brownfield Addition

## User Story

Como desenvolvedor,
Eu quero um middleware de erro global no servidor Hono,
Para que erros não capturados sejam tratados com respostas consistentes e logs adequados.

## Story Context

**Existing System Integration:**

- **Integrates with:** `apps/server/src/index.ts` — middleware chain do Hono
- **Technology:** Hono 4.8.2 (suporte nativo a `app.onError()`)
- **Follows pattern:** Hono já usa `app.use("*", logger())` e `app.use("*", cors())` — adicionar error handler
- **Touch points:** `apps/server/src/index.ts`

## Acceptance Criteria

**Functional Requirements:**

1. Middleware `onError` global registrado no app Hono
2. Erros conhecidos (Zod validation errors) retornam 400 com `{ error: string, details?: unknown }`
3. Erros de autenticação (Better-Auth) retornam 401/403
4. Erros desconhecidos retornam 500 com `{ error: "Internal server error" }` (sem vazar detalhes)
5. Log estruturado de todos os erros (método, path, status, mensagem, stack em dev)

**Integration Requirements:**

6. Endpoint `POST /ai` continua funcionando
7. Endpoints `/api/auth/*` do Better-Auth continuam funcionando
8. Health check `GET /` continua funcionando

**Quality Requirements:**

9. Testes unitários para o middleware de erro
10. Em desenvolvimento (`NODE_ENV=development`), stack trace incluído na resposta

## Technical Notes

- **Integration Approach:** Usar `app.onError((err, c) => {...})` do Hono — middleware nativo, zero dependências.
- **Existing Pattern Reference:** Hono docs para `onError`: https://hono.dev/docs/api/application#apponerror
- **Key Constraints:**
  - Better-Auth tem seu próprio tratamento de erro — não interferir
  - Não expor detalhes internos em produção (`NODE_ENV=production`)
  - Usar `console.error` para logs (sem dependência adicional de logger)

## Risk and Compatibility Check

- **Primary Risk:** `onError` pode capturar erros do Better-Auth e mudar comportamento esperado
- **Mitigation:** Verificar se o erro já tem status code definido (Better-Auth retorna 401/403 próprios) e respeitar
- **Rollback:** Remover `app.onError()` — Hono não tem error handler por padrão
