# Story 1.7: Criar .env.example para a Aplicação — Brownfield Addition

## User Story

Como novo desenvolvedor no projeto,
Eu quero um arquivo `.env.example` claro e completo,
Para que eu saiba exatamente quais variáveis de ambiente preciso configurar para rodar a aplicação.

## Story Context

**Existing System Integration:**

- **Integrates with:** Root do projeto, `apps/server/.env`, `apps/web/.env`
- **Technology:** dotenv (via `packages/env/`)
- **Follows pattern:** Root `.env.example` existe mas é para AIOX tooling — criar app-specific
- **Touch points:** `apps/server/.env.example`, `apps/web/.env.example`, opcionalmente root `.env.example` (atualizar ou complementar)

## Acceptance Criteria

**Functional Requirements:**

1. `apps/server/.env.example` criado com TODAS as variáveis necessárias:
   - `DATABASE_URL` com exemplo e explicação
   - `BETTER_AUTH_SECRET` com instrução de geração (mín 32 chars)
   - `BETTER_AUTH_URL` com default `http://localhost:3000`
   - `CORS_ORIGIN` com default `http://localhost:3001`
   - `AI_GATEWAY_API_KEY` com instrução de uso manual opcional
   - `VERCEL_OIDC_TOKEN` com instrução de uso via `vercel env pull`
   - `NODE_ENV` com default `development`
2. `apps/web/.env.example` criado com:
   - `VITE_SERVER_URL` com default `http://localhost:3000`

**Integration Requirements:**

3. Root `.env.example` atualizado com comentário claro diferenciando app vars vs AIOX tooling
4. `packages/env/src/server.ts` e `packages/env/src/web.ts` referenciados como fonte da verdade

**Quality Requirements:**

5. Exemplos usam valores válidos (URLs reais, não placeholders genéricos)
6. Documentação de como gerar `BETTER_AUTH_SECRET`

## Technical Notes

- **Integration Approach:** Criar arquivos `.env.example` em `apps/server/` e `apps/web/`. Atualizar root `.env.example` com seção separada.
- **Existing Pattern Reference:** `packages/env/src/server.ts` (16 lines) define exatamente quais vars são validadas.
- **Key Constraints:**
  - `BETTER_AUTH_SECRET` precisa de 32+ caracteres — sugerir `openssl rand -hex 32` ou `bun run --eval "crypto.randomBytes(32).toString('hex')"`
  - `SKIP_ENV_VALIDATION` mencionado como opt-in para Docker
