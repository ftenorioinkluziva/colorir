# Story 1.4: Error Boundary no Frontend — Brownfield Addition

## User Story

Como usuário,
Eu quero uma experiência controlada quando erros ocorrem na aplicação,
Para que eu não veja telas brancas ou crashes inesperados.

## Story Context

**Existing System Integration:**

- **Integrates with:** `apps/web/src/main.tsx` e `apps/web/src/routes/__root.tsx`
- **Technology:** React 19.2.6 (Error Boundary via class component ou `react-error-boundary`)
- **Follows pattern:** TanStack Router já tem `defaultPendingComponent` para loading — adicionar `defaultErrorComponent`
- **Touch points:** `apps/web/src/main.tsx`, `apps/web/src/routes/__root.tsx`

## Acceptance Criteria

**Functional Requirements:**

1. Error boundary global no root da aplicação React
2. UI amigável de erro com:
   - Mensagem "Algo deu errado"
   - Botão "Tentar novamente"
   - (Em desenvolvimento) detalhes do erro com stack trace
3. TanStack Router configurado com `defaultErrorComponent`
4. Erros em rotas individuais não quebram a aplicação inteira

**Integration Requirements:**

5. Tema dark/light respeitado na página de erro
6. TemaProvider continua funcionando acima do error boundary
7. Toaster (sonner) continua funcionando para erros não fatais

**Quality Requirements:**

8. Teste unitário para o ErrorBoundary component
9. Teste de integração simulando erro em rota

## Technical Notes

- **Integration Approach:** Duas camadas:
  1. Error boundary React clássico (ou `react-error-boundary`) no `main.tsx` envolvendo o Router
  2. `defaultErrorComponent` no TanStack Router (`__root.tsx`) para erros por rota
- **Existing Pattern Reference:** `apps/web/src/routes/__root.tsx` já define `defaultPendingComponent: Loader` — mesmo padrão para `defaultErrorComponent`.
- **Key Constraints:**
  - TemaProvider (`next-themes`) deve ficar FORA do error boundary para funcionar na página de erro
  - Usar componentes existentes de UI (`packages/ui/src/components/`)

## Decision

- Usar `react-error-boundary` (já é dependência via `@base-ui/react`? Verificar — se não, implementar ErrorBoundary class component para evitar dependência extra)
- Se for implementar classe: apenas ~30 linhas, sem dependências

## Definition of Done

- [x] ErrorBoundary component criado (class component ~40 linhas, zero dependências extra)
- [x] Error boundary global no `main.tsx` acima do Router

- [x] Página de erro amigável com theme-aware styling (Card + Button + AlertTriangle icon)
- [x] Botão "Tentar novamente" funcional (window.location.reload() + reset() por rota)

## Risk and Compatibility Check

- **Primary Risk:** Error boundary pode capturar erros do ThemeProvider se posicionado incorretamente
- **Mitigation:** ThemeProvider fica ACIMA do error boundary na árvore
- **Rollback:** Remover ErrorBoundary wrapper, remover `defaultErrorComponent`
