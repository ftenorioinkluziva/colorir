## User Story

Como usuário do Studio,
quero controlar parâmetros avançados de geração (aspect ratio, quantidade, tamanho, seed, e opções do provedor),
para que eu possa ajustar o resultado final conforme minha necessidade sem sair da plataforma.

## Acceptance Criteria

- [x] O schema de validação da API (`GenerateImageSchema`) aceita os novos parâmetros opcionais:
  - `aspectRatio`: string opcional (ex: "16:9", "1:1", "4:3", "3:2", "9:16")
  - `n`: número opcional (quantidade de imagens, respeitando limite do modelo)
  - `size`: string opcional (ex: "1024x1024", "512x512")
  - `seed`: número opcional (para reproducibilidade)
  - `providerOptions`: objeto opcional (opções específicas do provedor)
- [x] O endpoint `POST /api/generate-image` propaga os parâmetros para `generateImage()`
- [x] O fluxo de upload suporta múltiplas imagens retornadas (`n > 1`)
- [x] A resposta da API inclui os parâmetros utilizados para referência
- [x] O formulário do Studio expõe controles intuitivos para os parâmetros:
  - `aspectRatio`: seletor de proporção (ícone visual)
  - `n`: input number com limite documentado (max 3 para o modelo atual)
  - `size`: dropdown com tamanhos pré-definidos (ou "Automático")
  - `seed`: campo opcional (ajudar debug/reprodução)
  - Demais opções agrupadas em seção "Avançado" (collapsible)
- [x] Os valores default são aplicados quando o usuário não especifica nada (comportamento atual preservado)
- [x] O spike de validação (`validation/generate-image.ts`) também suporta os novos parâmetros

## Dev Notes

- Referências técnicas:
  - [AI SDK generateImage Reference](https://sdk.vercel.ai/docs/reference/ai-sdk-core/generate-image)
  - [AI Gateway Image Generation](https://vercel.com/docs/ai-gateway/capabilities/image-generation/ai-sdk)
- Parâmetros `generateImage` disponíveis: `aspectRatio`, `n`, `size`, `seed`, `providerOptions`
- O modelo atual via gateway define os limites: máximo 3 imagens por prompt
- `providerOptions` deve aceitar qualquer objeto, pois as opções variam por provedor
- A resposta atual `{ id, url, style, prompt, createdAt }` deve ser estendida para array de imagens quando `n > 1`
- Compatibilidade retroativa: requisições sem os novos parâmetros devem funcionar exatamente como antes

### Security Review

### Performance Considerations
Parameter propagation is optional — zero overhead when not used. Multi-image upload runs in parallel via `Promise.all`. RandomUUID suffix ensures filename uniqueness even for rapid multi-image generation.

### Architectural Observations
- **Response shape change**: The API response changed from `{ id, url }` to `{ images: [{ id, url }] }` — always returns an array for consistency. The frontend was updated in tandem, so no breakage. This deviates slightly from the Dev Notes suggestion to only extend "when n > 1", but the consistent array approach is a better design choice.
- **`providerOptions as never` cast**: Required for AI SDK type compatibility. Noted in Dev Log. Acceptable trade-off.
- **seed=0 edge case**: If user types "0" in seed field, `if (seed)` is falsy so it won't be sent. Edge case — seed=0 is valid but unlikely to be intentional.
