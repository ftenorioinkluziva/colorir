# Story 2.8: Migrar Geração de Imagem para generateText (Gemini)

**ID:** 2.8 | **Epic:** Epic 2: AI Studio + Galeria
**Order:** 13/13 | **Sprint:** 2 | **Points:** 5 | **Priority:** 🟠 High
**Status:** Ready for Review
**Source:** [docs/prd.md](../../../prd.md)

## Context

Após a implementação da story 2.7, foram realizados testes comparativos entre:

- **`generateImage`** com modelos de imagem dedicados (`google/imagen-4.0-*`, `openai/gpt-image-1`)
- **`generateText`** com `google/gemini-2.5-flash-image` (via `result.files`)

**Resultado:** O Gemini via `generateText` produziu qualidade superior para line-art B&W, com melhor aderência ao prompt estruturado e contornos mais limpos.

## Executor Assignment

- **executor:** @dev
- **quality_gate:** @qa
- **quality_gate_tools:** api_contract_review, visual_quality_review, validation_review

## User Story

Como usuário do Studio,
quero que a geração de imagens use `generateText` com `google/gemini-2.5-flash-image`,
para que eu obtenha line-art de qualidade superior mantendo os parâmetros avançados.

## Acceptance Criteria

- [x] O endpoint `POST /api/generate-image` usa `generateText` em vez de `generateImage`
- [x] A imagem gerada é extraída de `result.files` (filtrando por `mediaType.startsWith("image/")`)
- [x] O fluxo de upload e persistência permanece compatível (usa `uint8Array`)
- [x] A resposta da API mantém o mesmo contrato definido na 2.7 (`{ images: [{ id, url }] }`)
- [x] Os parâmetros avançados compatíveis com `generateText` são preservados: `seed`, `providerOptions`
- [x] Parâmetros de imagem não compatíveis (`aspectRatio`, `size`) são removidos do schema
- [x] O parâmetro `n` é removido (Gemini gera 1 imagem por chamada via text)
- [x] O prompt `buildLineArtPrompt` não inclui mais restrição "Generate one single image only" (desnecessária)
- [x] O spike de validação (`validation/generate-image.ts`) é atualizado para usar `generateText`
- [x] Testes de schema, rota e tipo passam

## Scope

- In scope: troca de `generateImage` → `generateText`, ajuste do schema Zod, atualização da rota, frontend e validação
- Out of scope: alteração no conteúdo do prompt de line-art, novo provider, redesign da galeria
