## 2. Recommended Model: AI Gateway image model

### 2.1 Rationale

- **AI Gateway routing:** model/provider selection stays behind the Vercel AI Gateway
- **Image generation support:** use the image-capable model configured in the gateway
- **Speed:** choose a low-latency image-capable model for the interactive flow
- **Quality:** prioritize clean B&W line-art suitable for coloring books
- **No provider SDK coupling:** the app depends on the gateway contract, not a provider SDK

### 2.2 Alternative Models Considered

| Model | Pros | Cons | Verdict |
|-------|------|------|---------|
| `openai/gpt-image-1` | Gateway-routed image generation | Depends on gateway availability and model support | ✅ **Recommended** |
| OpenRouter proxy models | Multiple providers via one key | Needs extra routing config, extra latency | ⏳ Future option |
| Other image-capable gateway models | Alternative capabilities | Validate output quality before adoption | 🔍 Monitor |

continuous black strokes on a white background.

### 4.1 Recommended Approach: `generateText` with Gemini Image Model

```ts
import { generateImage } from "ai";

const result = await generateImage({
  model: "openai/gpt-image-1",
  prompt: `${systemInstruction}\n\n${userPrompt}`,
});

// Extract image from response
for (const file of result.files) {
  if (file.mediaType?.startsWith("image/")) {
    // file.uint8Array -> buffer for MinIO upload
    // file.base64 -> alternate format
  }
}
```

**Why `generateText` over `generateImage`:**
- Simpler API surface for our use case
- Gemini Flash image output works naturally with `generateText`
- `generateImage` keeps the image-generation contract explicit and avoids provider SDK coupling

### 4.2 Prompt Construction
