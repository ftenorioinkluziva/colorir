## 2. Recommended Model: `google/gemini-2.5-flash-image`

### 2.1 Rationale

- **Already installed:** `@ai-sdk/google` v3.0.1 is in `package.json`
- **Image generation support:** Gemini 2.5 Flash supports native image generation via both:
  - `generateImage({ model: google.image('gemini-2.5-flash-image'), ... })`
  - `generateText({ model: 'google/gemini-2.5-flash-image', ... })` — images returned in `result.files`
- **Speed:** Flash tier is optimized for low-latency (target <10s per FR3)
- **Quality:** Produces clean B&W line-art suitable for coloring books
- **No additional packages needed:** `@ai-sdk/google` already covers this

### 2.2 Alternative Models Considered

| Model | Pros | Cons | Verdict |
|-------|------|------|---------|
| `google/gemini-2.5-flash-image` | Already installed, fast, native image gen | Requires `GOOGLE_GENERATIVE_AI_API_KEY` | ✅ **Recommended** |
| OpenRouter proxy models | Multiple providers via one key | Needs `@ai-sdk/openrouter`, extra latency | ⏳ Future option |
| `google/gemini-3-pro-image-preview` | Higher quality | Preview model, slower | 🔍 Monitor for GA |

## 4. Recommended API Implementation (for 2.2/2.3)

### 4.1 Recommended Approach: `generateText` with Gemini Image Model

```ts
import { generateText } from "ai";

const result = await generateText({
  model: "google/gemini-2.5-flash-image",
  prompt: `${systemInstruction}\n\n${userPrompt}`,
});

// Extract image from response
for (const file of result.files) {
  if (file.mediaType?.startsWith("image/")) {
    // file.uint8Array -> buffer for MinIO upload
    // file.base64 -> alternate format
  }
}
```

**Why `generateText` over `generateImage`:**
- Simpler API surface for our use case
- Gemini Flash image output works naturally with `generateText`
- `google.image('gemini-2.5-flash-image')` requires `google` import + `generateImage` — both work, but `generateText` with model string is more portable
